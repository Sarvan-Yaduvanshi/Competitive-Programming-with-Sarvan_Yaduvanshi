## 📖 About This Module

This module is the **most comprehensive Sorting & Searching resource** for competitive programmers. It covers **every single concept** from absolute basics to Grandmaster-level techniques — all in production-quality, heavily-commented C++17 code.

**What makes this different?**
- 🧠 **Theory-first approach** — Every algorithm explained with intuition, proof sketches, and visual dry runs
- 💻 **Complete working code** — Every concept has runnable implementations with test cases
- 📊 **Complexity tables** — Best/Average/Worst case for every algorithm at a glance
- 🏆 **100+ practice problems** — Curated from Codeforces, LeetCode, CSES, SPOJ, AtCoder
- 🔄 **Progressive difficulty** — From "What is sorting?" to "Aliens trick + Parallel binary search"

### Searching Algorithms

```
┌──────────────────────┬──────────────┬───────────┬──────────────────────────┐
│ Algorithm            │ Time         │ Space     │ Prerequisite             │
├──────────────────────┼──────────────┼───────────┼──────────────────────────┤
│ Linear Search        │ O(n)         │ O(1)      │ None                     │
│ Binary Search        │ O(log n)     │ O(1)      │ Sorted array             │
│ Ternary Search       │ O(log n)     │ O(1)      │ Unimodal function        │
│ Interpolation Search │ O(log log n) │ O(1)      │ Uniformly distributed    │
│ Exponential Search   │ O(log n)     │ O(1)      │ Sorted (unbounded)       │
│ Jump Search          │ O(√n)        │ O(1)      │ Sorted array             │
│ Staircase Search     │ O(n + m)     │ O(1)      │ Row+Col sorted matrix    │
│ BS on Answer         │ O(log V · C) │ O(1)      │ Monotonic feasibility    │
│ Parallel BS          │ O(n log²n)   │ O(n)      │ Offline queries          │
│ Fractional Cascading │ O(log n + k) │ O(n)      │ Multiple sorted lists    │
└──────────────────────┴──────────────┴───────────┴──────────────────────────┘
  V = value range, C = cost of feasibility check
```

## ⚡ Quick Start Guide

### Clone the Repository

```bash
git clone https://github.com/Sarvan-Yaduvanshi/Competitive-Programming-with-Sarvan_Yaduvanshi.git
cd Competitive-Programming-with-Sarvan_Yaduvanshi/05_Sorting_And_Searching_Techniques
```

### Compile & Run Any Chapter

```bash
# Compile a specific chapter
g++ -std=c++17 -O2 -o ch01 01_Sorting_Basics_And_Fundamentals/Sorting_Basics_And_Fundamentals.cpp
./ch01

# Compile with debug flags
g++ -std=c++17 -g -fsanitize=address -o ch07_debug 07_Binary_Search_On_Answer/Binary_Search_On_Answer.cpp
./ch07_debug

# Compile all chapters at once
for dir in */; do
  cpp=$(find "$dir" -name "*.cpp" | head -1)
  [ -n "$cpp" ] && g++ -std=c++17 -O2 -o "/tmp/$(basename $dir)" "$cpp" && echo "✅ $dir"
done
```

### Using CMake

```bash
cd ../..
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make
```

## 💡 Key Insights & Tips

### 🔑 The 5 Golden Rules of Sorting in CP

1. **Always consider sorting first** — If the problem doesn't require preserving order, sorting is almost always beneficial
2. **Use `sort()` not `stable_sort()`** — Unless stability matters; `sort()` is 2-3× faster
3. **Never use `<=` in comparators** — Violates strict weak ordering → undefined behavior
4. **Coordinate compress before BIT/Segment Tree** — Map values to [0, n) range
5. **Binary search on answer when direct computation is hard** — If you can verify, you can search

### 🔑 The 5 Golden Rules of Binary Search

1. **Identify the monotonic property** — BS only works when predicate goes F→T or T→F
2. **Always use `lo + (hi - lo) / 2`** — Prevents integer overflow
3. **Know your template** — `lo < hi` with `hi = mid` vs `lo <= hi` with `hi = mid - 1`
4. **Handle floating-point BS** with fixed iterations (100-200), not epsilon comparison
5. **When stuck, think "can I binary search on the answer?"** — This solves 30% of hard problems
